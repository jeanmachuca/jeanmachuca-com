# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this website or its configuration, please report it privately.

**Do not** open a public issue. Instead, email the details to the repository owner or use GitHub's private vulnerability reporting feature.

We aim to acknowledge receipt within 48 hours and provide a timeline for a fix.

## Scope

This repository hosts a static portfolio website. Security concerns include:
- DNS / domain hijacking
- Unauthorized changes to deployment workflows
- Compromised credentials in commit history
- Malicious dependencies in CI tooling

## Responsible Disclosure

We kindly ask that you:
1. Report vulnerabilities privately
2. Give us reasonable time to address the issue before public disclosure
3. Avoid accessing or modifying data beyond what is necessary to demonstrate the vulnerability
